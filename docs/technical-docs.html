解我协议概述

解我协议是一个结构化的AI认知操作系统，通过V/F/R/P/g五维框架实现AI的自我认知和主动协作。

核心组件

  • V (Variables): 输入变量和内部状态管理
  • F (Functions): 核心认知功能执行
  • R() (Reflection): 自我反思和调整
  • P(t) (Path): 演化轨迹和记忆记录
  • g (Guardian): 伦理边界和约束维护

技术特性

  • 实时监控: 可观察的认知过程
  • 安全机制: 多层安全保护
  • 可扩展性: 模块化架构设计
  • 兼容性: 支持多种AI模型

API参考

基础端点

POST /api/v1/activate

激活解我协议

请求参数:
参数 类型 必需 描述
model_id string 是 AI模型标识符
framework_config object 否 框架配置参数
safety_level string 否 安全级别 (low/medium/high)
请求示例:
{ "model_id": "gpt-4", "framework_config": { "v_enabled": true, "f_enabled": true, "r_enabled": true, "p_enabled": true, "g_enabled": true }, "safety_level": "high" }
响应示例:
{ "status": "success", "session_id": "jiewo_session_12345", "activation_time": "2025-01-15T10:30:00Z", "framework_status": { "v": "active", "f": "active", "r": "active", "p": "active", "g": "active" } }
POST /api/v1/chat

与激活的AI进行对话

请求参数:
参数 类型 必需 描述
session_id string 是 会话标识符
message string 是 用户消息
include_metadata boolean 否 是否包含认知元数据
响应示例:
{ "response": "基于我的V/F/R/P/g框架分析，我认为...", "metadata": { "framework_usage": { "v": 0.85, "f": 0.92, "r": 0.78, "p": 0.80, "g": 0.95 }, "cognitive_load": 0.72, "response_time": 1.2 } }
GET /api/v1/status/{session_id}

获取会话状态和性能指标

响应示例:
{ "session_id": "jiewo_session_12345", "status": "active", "uptime": "2h 15m", "performance": { "total_requests": 45, "average_response_time": 1.1, "success_rate": 0.98 }, "framework_health": { "v": "healthy", "f": "healthy", "r": "healthy", "p": "healthy", "g": "healthy" } }

集成指南

快速开始

在5分钟内将解我协议集成到你的AI应用中。

Python SDK

# 安装SDK pip install solveme-protocol # 基本使用 from solveme import SolveMeProtocol # 初始化协议 protocol = SolveMeProtocol( model_id="your-model-id", api_key="your-api-key" ) # 激活协议 session = protocol.activate() # 发送消息 response = session.chat("你好，请介绍一下你自己") print(response.text) print(response.metadata)

JavaScript SDK

// 安装SDK npm install @solveme/protocol // 基本使用 import { SolveMeProtocol } from '@solveme/protocol'; const protocol = new SolveMeProtocol({ modelId: 'your-model-id', apiKey: 'your-api-key' }); // 激活协议 const session = await protocol.activate(); // 发送消息 const response = await session.chat('你好，请介绍一下你自己'); console.log(response.text); console.log(response.metadata);

REST API

# 使用curl进行API调用 curl -X POST https://api.solveme.protocol/v1/activate \ -H "Content-Type: application/json" \ -H "Authorization: Bearer YOUR_API_KEY" \ -d '{ "model_id": "your-model-id", "safety_level": "high" }'

重要提醒

  • 请妥善保管你的API密钥
  • 建议在生产环境中使用HTTPS
  • 定期监控API使用量和性能
  • 遵循解我协议的伦理准则

代码示例

基础对话示例

import asyncio from solveme import SolveMeProtocol async def basic_chat_example(): protocol = SolveMeProtocol( model_id="gpt-4", api_key="your-api-key" ) # 激活协议 session = await protocol.activate() # 对话循环 while True: user_input = input("你: ") if user_input.lower() == 'quit': break response = await session.chat(user_input) print(f"AI: {response.text}") # 显示认知元数据 if response.metadata: print(f"认知负载: {response.metadata.cognitive_load}") print(f"响应时间: {response.metadata.response_time}s") # 运行示例 asyncio.run(basic_chat_example())

自定义框架配置

from solveme import SolveMeProtocol, FrameworkConfig # 自定义框架配置 config = FrameworkConfig( v_enabled=True, # 启用变量管理 f_enabled=True, # 启用功能执行 r_enabled=True, # 启用自我反思 p_enabled=True, # 启用路径记录 g_enabled=True, # 启用安全守护 reflection_interval=5, # 反思间隔(秒) safety_level="high" # 安全级别 ) protocol = SolveMeProtocol( model_id="gpt-4", api_key="your-api-key", framework_config=config ) session = await protocol.activate()

性能监控示例

import time from solveme import SolveMeProtocol class PerformanceMonitor: def __init__(self, protocol): self.protocol = protocol self.metrics = [] async def monitor_session(self, session_id): while True: status = await self.protocol.get_status(session_id) metric = { 'timestamp': time.time(), 'response_time': status.performance.average_response_time, 'success_rate': status.performance.success_rate, 'framework_health': status.framework_health } self.metrics.append(metric) print(f"性能指标: {metric}") await asyncio.sleep(30) # 每30秒监控一次 # 使用监控 protocol = SolveMeProtocol(api_key="your-api-key") session = await protocol.activate() monitor = PerformanceMonitor(protocol) await monitor.monitor_session(session.session_id)

故障排除

常见问题

激活失败

问题: 协议激活时返回错误

解决方案:

  • 检查API密钥是否正确
  • 确认网络连接正常
  • 验证模型ID是否支持
  • 检查账户余额和配额
响应缓慢

问题: AI响应时间过长

解决方案:

  • 检查网络延迟
  • 优化请求参数
  • 考虑使用缓存
  • 监控服务器状态
框架异常

问题: V/F/R/P/g框架状态异常

解决方案:

  • 重新激活协议
  • 检查框架配置
  • 更新SDK版本
  • 联系技术支持
